# Air Mouse

Система «аэромыши»: смартфон как пульт с гироскопом → Android TV с курсором.

Два приложения в одном Gradle-проекте + общий модуль бинарного протокола:

| Модуль | Приложение | Назначение |
|---|---|---|
| `protocol` | библиотека | бинарный UDP-протокол (общий для клиента и сервера) |
| `phone` | `app-release.apk` | клиент: гироскоп → sensor fusion → UDP-команды |
| `tv` | `tv-release.apk` | сервер: UDP-приём → overlay-курсор → эмуляция нажатий |

## Архитектура

```
 Смартфон (phone)                           Android TV (tv)
 ┌────────────────────────┐                 ┌──────────────────────────┐
 │ Gyro+Accel (60 Гц)     │                 │ UdpServer :5005          │
 │   ↓                    │                 │   ↓ decode               │
 │ MotionEngine           │   UDP (binary)  │ handlePacket             │
 │   Deadzone → EMA       │  ─────────────▶ │   ↓                      │
 │   → Complementary      │   MOVE/TAP/...  │ CursorState + CursorOverlay
 │   → px = ω·dt·sens     │                 │   ↓ dispatchGesture      │
 │   ↓                    │                 │ клик/скролл в приложениях│
 │ UdpTransport ──────────┤                 │                          │
 │ Discovery (broadcast)  │ ◀── ANNOUNCE ── │ DiscoveryResponder       │
 └────────────────────────┘                 └──────────────────────────┘
```

### Почему так

- **UDP, не TCP.** Поток координат: потеря пакета = потеря микро-движения, а не «заикание» от ретрансмиссий TCP.
- **Относительные дельты (MOVE), не абсолютные координаты.** Stateless-протокол: сервер сам накапливает позицию и клампит по границам. Нет рассинхрона при потере пакетов.
- **Бинарный формат (little-endian).** `[type:1u8][payload]`. Легче и быстрее JSON, что критично для пинга.
- **Sensor fusion на клиенте.** Гироскоп точен в краткосрочной перспективе, но дрейфует; акселерометр шумит, но даёт вектор гравитации. Комбинация + EMA + deadzone убирает дрожание и дрейф. Полный yaw-дрейф без магнитометра не убирается, но движение курсора относительное, а накопленный сдвиг сбрасывается кнопкой «Сброс центра».
- **AccessibilityService на ТВ.** Единственный способ без Root рисовать курсор поверх окон (`TYPE_ACCESSIBILITY_OVERLAY`) и эмулировать нажатия (`dispatchGesture`). Не требует `SYSTEM_ALERT_WINDOW`.

## Сборка APK

Проект собирается в Android Studio (минимум — установленная Android SDK).

> ⚠️ В репозитории намеренно **нет** `gradle-wrapper.jar` и `local.properties`.
> Android Studio сгенерирует их при первом открытии проекта.

1. Откройте папку `airmouse/` в Android Studio.
2. Если студия предложит «Use Gradle from: gradle-wrapper.jar» — согласитесь
   (она создаст wrapper), либо выберите свой локальный Gradle 8.7.
3. Дождитесь окончания Gradle Sync.
4. Для сборки обоих APK выберите слева нужный модуль (`phone` или `tv`),
   затем **Build → Build Bundle(s) / APK(s) → Build APK(s)**.
5. Готовые файлы:
   - `phone/build/outputs/apk/debug/phone-debug.apk`
   - `tv/build/outputs/apk/debug/tv-debug.apk`

Для release-сборки (с подписью): создайте signing config в
`phone/build.gradle.kts` и `tv/build.gradle.kts`, затем `Build → Generate Signed APK`.

Версии стека: AGP 8.5.2, Kotlin 1.9.24, Gradle 8.7, JDK 17, compileSdk 34, minSdk 24.

## Установка и настройка

### На Android TV (`tv-debug.apk`)
1. Установите APK (через `adb install` или USB).
2. Откройте приложение — вы увидите свой IP-адрес и порт 5005.
3. Нажмите **Включить службу** → в системных настройках включите
   «Air Mouse — Сервер» в разделе **Настройки → Специальные возможности**.
4. Статус в приложении должен смениться на «Включена — приём команд активен».

### На смартфоне (`phone-debug.apk`)
1. Установите APK.
2. Убедитесь, что телефон и ТВ в одной Wi-Fi сети.
3. Введите IP ТВ вручную и нажмите «Подключиться», либо «Поиск ТВ»
   (UDP-broadcast автообнаружение) и выберите найденное устройство.
4. Держите телефон как пульт — движение кистью перемещает курсор.

## Управление

| Кнопка | Действие |
|---|---|
| Движение кистью | перемещение курсора |
| ЛКМ | короткий клик в точке курсора |
| ПКМ | длинное нажатие (контекстное меню) |
| Назад / Домой | системные кнопки ТВ |
| ↑ / ↓ | скролл списков |
| Сброс центра | сброс курсора в центр экрана (компенсация дрейфа) |
| Настройки | чувствительность, сглаживание (α), мёртвая зона |

## Тестирование (по этапу 5 плана)

| Тест | Что проверить | Какой компонент отвечает |
|---|---|---|
| **Дрейф** | Положить телефон на стол → курсор на ТВ стоит | `Deadzone` |
| **Дрожание** | Держать неподвижно → курсор стабилен | `EmaFilter` (α≈0.2) |
| **Задержка** | Движение синхронно, без лагов | throttle 60 Гц + binary-протокол |
| **Клик/скролл** | Срабатывают в YouTube, Кинопоиске, браузере | `GestureExecutor.dispatchGesture` |
| **Системные кнопки** | BACK/HOME работают глобально | `performGlobalAction` |

Если курсор **дрейфует** в покое — увеличьте «Мёртвую зону» в настройках клиента.
Если курсор **дрожит** в руке — увеличьте «Сглаживание».
Если курсор **медленный** — поднимите «Чувствительность».

## Структура проекта

```
airmouse/
├── settings.gradle.kts, build.gradle.kts, gradle.properties
├── gradle/libs.versions.toml, gradle/wrapper/gradle-wrapper.properties
├── protocol/   — бинарный протокол (Packet, PacketCodec, Net)
├── phone/      — клиент: sensor/, net/, MainActivity, SettingsActivity
└── tv/         — сервер: AirMouseAccessibilityService, CursorOverlay,
                  GestureExecutor, CursorState, net/UdpServer, net/DiscoveryResponder
```

## Возможные расширения

- Foreground-сервис на телефоне (сенсоры работают в фоне, экран выключен).
- Магнитометр для полного убирания yaw-дрейфа (алгоритм Madgwick/Mahony).
- Compose-UI вместо Views.
- Авто-переподключение при потере связи.
- mDNS/NSD вместо UDP-broadcast (актуально для роутеров, режущих broadcast).
